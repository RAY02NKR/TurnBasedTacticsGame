# Turn-Based Tactics Battle

## 作品概要

**Turn-Based Tactics Battle** は、6×6のグリッド上で味方ユニットの移動経路・使用スキル・対象を事前に計画し、ターン開始後に敵味方の行動を速度順で解決するターン制タクティクスゲームです。

単純なコマンド選択ではなく、**「事前に立てた行動計画が、実行フェーズでどのように解決されるか」** をゲームの中心にしています。プレイヤーは敵の位置や行動予測、味方の移動先、スキル射程、状態異常を確認しながら、ターン開始前に行動を組み立てます。

## ゲームの基本ルール

### 勝利条件

敵ユニットをすべて倒すと勝利です。

### 敗北条件

味方ユニットがすべて倒されると敗北です。

### ターンの流れ

1. ターン開始
2. 計画フェーズ
3. 味方ユニットの移動経路を設定
4. 使用スキルを選択
5. 対象・位置・方向を指定
6. レバー、スペースキー、またはEnterキーで戦闘開始
7. 敵味方の行動を速度順で解決
8. ターン終了処理
9. 勝敗がついていなければ次ターンへ

## 操作方法

| 操作 | 内容 |
| --- | --- |
| 味方ユニットを短く左クリック | スキル選択パネルを開く |
| 味方ユニットを長押し、またはドラッグ | 移動経路を設定 |
| スキルボタンをクリック | 使用するスキルを選択 |
| ハイライトされたマス・対象・方向をクリック | スキル対象を確定 |
| 右クリック | 対象選択や選択状態をキャンセル |
| スペース、Enterキー、またはレバー操作 | 計画した行動を実行 |
| ユニットにカーソルを合わせる | HP・ステータス・予定行動を表示 |

## 計画システム

本作では、味方ユニットごとに以下の内容を事前に設定します。

- 移動経路
- 使用スキル
- スキルの対象、位置、または方向

行動はターン開始後に速度順で解決されます。移動先が他ユニットなどで塞がっている場合は、計画経路上で到達可能な位置まで移動します。移動後にスキルの射程条件を満たしていれば、その位置からスキルを使用します。

また、一度設定した行動は再調整できます。移動経路を変更してからスキルを選び直すことも可能です。

## 味方ユニット

### Knight

近接攻撃、火傷、ノックバック、カウンターを扱う前衛ユニットです。

| スキル | 種別 | 説明 |
| --- | --- | --- |
| 火炎斬 | 通常攻撃 | 近距離の敵を攻撃し、火傷を付与します。 |
| シールドバッシュ | アクティブ | 近距離の敵を攻撃し、直線方向にノックバックさせます。壁や障害物にぶつかると追加ダメージが発生します。 |
| カウンター | アクティブ | 自身にカウンターを付与します。被ダメージ時、攻撃者に反撃ダメージを与えます。 |
| 剣の流れ | アクティブ | 直線方向の敵に攻撃し、火傷を火傷-延焼へ変換します。 |

### Maestro

タレット召喚と充電を使う支援・遠距離型ユニットです。

| スキル | 種別 | 説明 |
| --- | --- | --- |
| 電気ハンマー | 通常攻撃 | 指定位置の敵を攻撃し、自身に充電を付与します。 |
| タレット召喚 | アクティブ | 指定位置に自動攻撃するタレットを召喚します。 |
| タレット強化 | アクティブ | 攻撃しつつ充電を付与し、次ターンにタレットを強化します。 |
| 電磁砲 | アクティブ | 充電を消費し、消費した充電数に応じて連続ダメージを与えます。 |

### Mage

火傷・出血・凍結などの状態異常を使い分ける魔法系ユニットです。継続ダメージや速度低下を与えることで、次ターン以降の行動計画を立てやすくする役割を持っています。

| スキル | 種別 | 説明 |
| --- | --- | --- |
| 攻撃 | 通常攻撃 | 敵単体に攻撃します。 |
| 血液操作 | アクティブ | 敵にダメージを与え、出血を付与します。出血中の敵は、移動するたびにダメージを受けます。 |
| 炎魔法 | アクティブ | 敵にダメージを与え、火傷を付与します。火傷はターン終了時にダメージを与えます。 |
| 氷魔法 | アクティブ | 敵にダメージを与え、凍結系の状態異常を付与します。敵の速度低下や追加ダメージにつながります。 |

### Turret

Maestroが召喚する自動行動ユニットです。敵を自動で狙い、通常攻撃を行います。

### Enemy

現在の敵ユニットは、味方へ接近し、射程内に入ると攻撃します。敵の行動はターン開始時に決定され、そのターン中は再計算されません。

## 状態異常・特殊効果

| 状態異常 | 内容 |
| --- | --- |
| 火傷 | ターン終了時にスタック数に応じたダメージを与えます。 |
| 火傷-延焼 | 火傷から派生する状態異常です。ターン終了時にダメージを与え、スタックの一部を次ターンへ引き継ぎます。 |
| 出血 | ユニットが1マス移動するたびにダメージを受けます。ノックバックなどの強制移動でも発動します。 |
| 充電 | Maestroが電磁砲を使うために必要なリソースです。 |
| カウンター | 被ダメージ時、攻撃者に反撃ダメージを与えます。 |
| スロー | 次ターンの速度を低下させます。 |
| 凍結系 | 凍結、凍傷、氷晶化の派生状態を持つ状態異常です。 |
| タレット強化 | タレットの攻撃力を一時的に上昇させます。 |
| フリーステップ | 一時的に移動可能距離を大きく伸ばします。 |

## 設計上の工夫

### 1. ScriptableObjectによるデータ駆動

ユニット、スキル、状態異常をScriptableObjectとして管理しています。スキル威力、射程、クールタイム、付与する状態異常などをコードを書き換えずに調整できます。

### 2. Stateパターンによる進行管理

バトルの流れを `IBattleState` を使って状態ごとに分離しています。初期化、ターン開始、計画、行動解決、勝敗判定を独立した処理として管理することで、処理の流れを把握しやすくしています。

### 3. スキル効果の分離

スキルは、ダメージ、状態異常付与、ノックバック、召喚、充電消費などの効果を組み合わせて構成しています。新しい効果を追加する場合も、既存スキル全体を書き換えずに拡張しやすい構造を意識しました。

### 4. UIによる情報提示

ターン制ゲームでは、プレイヤーが現在の状況を理解できることが重要だと考え、HP、状態異常、予定行動、ダメージ、行動中ユニットを画面上で確認できるようにしました。

## 制作目的・意識したこと

この作品は、UnityとC#を用いた個人制作作品として、ターン制バトルの設計・実装力を示すために制作しました。

制作のきっかけは、「事前に計画を立て、その結果を見守る」タイプのゲームに面白さを感じたことです。現実でも、自分で立てた予定が思い通りに進むと達成感があり、失敗した場合でも原因を考えて次に活かすことができます。この「計画を立てる楽しさ」と「結果を見て改善する楽しさ」を、ゲームの中心にしたいと考えました。

そのため、本作ではプレイヤーがターン開始前に味方ユニットの移動やスキルをまとめて計画し、その後に敵味方の行動が順番に解決される形にしています。敵の位置や行動順、味方の移動先、スキルの射程、状態異常を踏まえて行動を組み立てることで、単に攻撃して敵を倒すだけではない戦闘を目指しました。

また、この仕組みを成立させるには盤面全体の把握が重要だと考え、アクションゲームや横スクロールゲームではなく、見下ろし型のターン制タクティクスゲームにしました。プレイヤーが「どこに移動するか」「どの順番で行動するか」「状態異常をどう活かすか」を考えられるようにすることを意識しています。
