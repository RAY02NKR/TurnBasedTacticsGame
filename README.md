# Turn-Based Tactics Battle

## 概要

**Turn-Based Tactics Battle** は、6×6のグリッド上で味方ユニットの移動経路・使用スキル・対象を事前に計画し、ターン開始後に敵味方の行動を速度順で解決するターン制タクティクスゲームです。

プレイヤーは各ターンの計画フェーズで、味方ユニットの移動とスキルを自由な順番で設定します。行動を確定すると、敵味方の移動・攻撃・状態異常・召喚ユニットの行動が順番に実行されます。

この制作物では、単純なコマンド選択ではなく、**「事前に立てた行動計画が、盤面上でどのように解決されるか」** をゲームの中心に置いています。

## ゲームの基本ルール

### 勝利条件

敵ユニットをすべて倒すと勝利です。

### 敗北条件

味方ユニットがすべて倒されると敗北です。

### ターンの流れ

1. ターン開始
2. 計画フェーズ
3. 味方ユニットの移動経路を設定
4. 使用スキルを選択
5. 対象・位置・方向を指定
6. レバーを引いて戦闘開始
7. 敵味方の行動を速度順で解決
8. ターン終了処理
9. 勝敗がついていなければ次ターンへ

## 操作方法

| 操作                    | 内容               |
| --------------------- | ---------------- |
| 味方ユニットを短く左クリック        | スキル選択パネルを開く      |
| 味方ユニットを長押し、またはドラッグ    | 移動経路を設定          |
| スキルボタンをクリック           | 使用するスキルを選択       |
| ハイライトされたマス・対象・方向をクリック | スキル対象を確定         |
| 右クリック                 | 対象選択や選択状態をキャンセル  |
| スペースまたはEnterキー            | 計画した行動を実行        |
| ユニットにカーソルを合わせる        | HP・ステータス・予定行動を表示 |

## 計画システム

本作では、味方ユニットごとに以下の内容を事前に設定します。

* 移動経路
* 使用スキル
* スキルの対象、位置、または方向

行動はターン開始後に速度順で解決されます。
移動先が塞がっている場合は、計画経路上で到達可能な位置まで移動します。
移動後にスキルの射程条件を満たしていれば、その位置からスキルを使用します。

また、一度設定した行動は再調整できます。
移動経路を変更してからスキルを選び直すことも可能です。

## 味方ユニット

### Knight

近接攻撃と火傷、ノックバック、カウンターを扱う前衛ユニットです。

| スキル      | 種別    | 説明                                                 |
| -------- | ----- | -------------------------------------------------- |
| 火炎斬      | 通常攻撃  | 近距離の敵を攻撃し、火傷を付与します。                                |
| シールドバッシュ | アクティブ | 近距離の敵を攻撃し、直線方向にノックバックさせます。壁や障害物にぶつかると追加ダメージが発生します。 |
| カウンター    | アクティブ | 自身にカウンターを付与します。被ダメージ時、攻撃者に反撃ダメージを与えます。             |
| 剣の流れ     | アクティブ | 直線方向の敵に攻撃し、火傷を火傷-延焼へ変換します。                         |

### Maestro

タレット召喚と充電を使う支援・遠距離型ユニットです。

| スキル    | 種別    | 説明                             |
| ------ | ----- | ------------------------------ |
| 電気ハンマー | 通常攻撃  | 指定位置の敵を攻撃し、自身に充電を付与します。        |
| タレット召喚 | アクティブ | 指定位置に自動攻撃するタレットを召喚します。         |
| タレット強化 | アクティブ | 攻撃しつつ充電を付与し、次ターンにタレットを強化します。   |
| 電磁砲    | アクティブ | 充電を消費し、消費した充電数に応じて連続ダメージを与えます。 |

### Turret

Maestroが召喚する自動行動ユニットです。
敵を自動で狙い、通常攻撃を行います。

### Enemy

現在の敵ユニットは、味方へ接近し、射程内に入ると攻撃します。
敵の行動はターン開始時に決定され、そのターン中は再計算されません。

## 状態異常・特殊効果

| 状態異常    | 内容                                                 |
| ------- | -------------------------------------------------- |
| 火傷      | ターン終了時にスタック数に応じたダメージを与えます。                         |
| 火傷-延焼   | 火傷から派生する状態異常です。ターン終了時にダメージを与え、スタックの一部を次ターンへ引き継ぎます。 |
| 出血      | ユニットが1マス移動するたびにダメージを受けます。ノックバックなどの強制移動でも発動します。     |
| 充電      | Maestroが電磁砲を使うために必要なリソースです。                        |
| カウンター   | 被ダメージ時、攻撃者に反撃ダメージを与えます。                            |
| スロー     | 次ターンの速度を低下させます。                                    |
| 凍結系     | 凍結、凍傷、氷晶化の派生状態を持つ状態異常です。                           |
| タレット強化  | タレットの攻撃力を一時的に上昇させます。                               |
| フリーステップ | 一時的に移動可能距離を大きく伸ばします。                               |

## 設計上の工夫

### ScriptableObjectによるデータ駆動

ユニット、スキル、状態異常をScriptableObjectとして管理しています。
これにより、スキル威力、射程、クールタイム、付与する状態異常などをコードを書き換えずに調整できます。

### Stateパターンによる進行管理

バトルの流れを `IBattleState` を使って状態ごとに分離しています。
これにより、初期化、ターン開始、計画、行動解決、勝敗判定を独立した処理として管理しています。

### スキル効果の分離

スキルは、ダメージ、状態異常付与、ノックバック、召喚、充電消費などの効果を組み合わせて構成しています。
新しい効果を追加する場合も、既存スキル全体を書き換えずに拡張しやすい構造にしています。

### UIによる情報提示

ターン制ゲームでは、プレイヤーが現在の状況を理解できることが重要だと考え、HP、状態異常、予定行動、ダメージ、行動中ユニットを画面上で確認できるようにしました。

## 制作を通して意識したこと

この制作では、単に動く戦闘を作るだけでなく、後からスキルや状態異常を追加しやすい構造を意識しました。

特に、状態異常やスキル効果は仕様変更が起きやすい部分なので、ScriptableObjectと効果クラスを分けることで、データ調整と処理追加をしやすくしています。

また、ターン制ゲームではプレイヤーが「なぜその結果になったのか」を理解できることが重要だと考え、HPバー、ダメージ表示、予定行動、ステータス表示など、状況を確認するためのUIも重視しました。
